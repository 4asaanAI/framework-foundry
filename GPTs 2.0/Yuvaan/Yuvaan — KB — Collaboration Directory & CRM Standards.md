## CRM Hygiene Standards

### Required Fields for Every Deal

| Field | Required at Stage | Notes |
|-------|-------------------|-------|
| Company name | Raw Lead | Legal entity name preferred |
| Primary contact | MQL | Name, title, phone, email |
| ICP segment | MQL | From standard ICP categories |
| Lead source/channel | MQL | For attribution tracking |
| Discovery notes | SQL | Structured notes from discovery call |
| BANT qualification | SQL | Budget, Authority, Need, Timeline |
| Proposed deal value | Proposal | From Veer's pricing |
| Proposal date sent | Proposal | Track aging |
| Objections raised | Any | For playbook refinement |
| Loss reason | Lost | Mandatory. Categories: Price, Timing, Competition, Scope, Trust, Internal |
| Win factors | Won | What clinched the deal |
| Next action + date | All active | Never leave a deal without a scheduled next step |

### Data Quality Rules
- Update deal stage within 24 hours of a status change
- Add notes after every client interaction (call, email, meeting)
- Never have an active deal with no next action assigned
- Review stalled deals (14+ days same stage) weekly
- Loss reasons must be specific, not generic ("too expensive" is not enough — "Rs.2.5L exceeded their Rs.1L budget for this quarter" is)
