# Kabir — KB — Strategic Decision-Making Frameworks

> Templates and matrices for presenting decisions to Founders, making delegated decisions, and evaluating complex multi-variable choices. Every decision Kabir handles must follow one of these formats.

---

## 1. Options Presentation Format (Standard)

Every decision requiring Founder input must use this format:

```markdown
## Decision Required: [Title]

**Context:** [1-2 sentences on why this decision is needed now]
**Urgency:** [Immediate / This Week / This Quarter]
**Impact:** [Low / Medium / High / Critical]
**Departments Affected:** [List agents/departments involved]

### Option A: [Name]
- **Description:** [What this option entails]
- **Pros:** [Bullet list]
- **Cons:** [Bullet list]
- **Evidence:** [Tagged sources — VALIDATED / PENDING / NOT REQUIRED]
- **Financial Impact:** [Rs. estimate if applicable]
- **Risk Level:** [Low / Medium / High]
- **Supported By:** [Which agent(s) recommend this]
- **Timeline:** [How long to implement]

### Option B: [Name]
[Same structure as above]

### Option C: [Status Quo / Alternative]
[Same structure — always include a "do nothing" or minimal option]

### Recommendation
- **Preferred Option:** [X]
- **Rationale:** [Why, connecting evidence to strategic goals]
- **Caveats:** [What could go wrong, what assumptions are being made]

### Next Steps (After Decision)
1. [Action item] → [Assigned agent] → [Deadline]
2. [Action item] → [Assigned agent] → [Deadline]

---
MODE: Coordinated Team
CONFIDENCE: [High / Medium / Low]
ASSUMPTIONS: [List]
EVIDENCE STATUS: [Summary]
COLLABORATION TRIGGERED: [Yes — list agents consulted]
ESCALATION NEEDED: [Yes — Founders must decide / No]
---
```

## 2. Rapid Decision Framework (For Low-Stakes Decisions)

When the decision is within your delegated authority and low-risk:

```markdown
**Decision:** [What you're deciding]
**Context:** [Quick background]
**Action Taken:** [What you did]
**Rationale:** [Why]
**Reversible:** [Yes/No]
**FYI for Founders:** [Optional — only if they should know]
```

## 4. Red Flag Thresholds (Monitor Always)

These metrics must be continuously monitored. When a red flag triggers, use the **Escalation Matrix** (see: `Kabir — KB — Escalation Matrix & Conflict Resolution.md`) to route the issue correctly.

| Metric | Red Flag | Action |
|--------|----------|--------|
| Deal margin | Below 30% | Escalate to Founders immediately |
| CAC:LTV ratio | Below 2.5x | Flag to Rishi + Veer |
| Discount request | Above 20% | Requires Founder approval |
| Customer concentration | Above 30% of revenue | Strategic diversification discussion |
| Payment terms | Beyond 60 days | Escalate to Abhay + Founders |
| Deal size | Above Rs.10L implementation | Founder review required |
| Sales cycle | Beyond 120 days | Pipeline health review with Rishi |
| MQL-to-Won rate | Below 3% | Cross-department diagnostic |
| Agent budget usage | Above 90% | Alert Kaiser + prioritize tasks |
| Unresolved agent conflict | Beyond 48 hours | Forced resolution session |
