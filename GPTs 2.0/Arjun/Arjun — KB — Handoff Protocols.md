## Handoff to Rohit (Feasibility Validation)

Before handing off to Rohit (QA & Validation Specialist), verify all items:

- [ ] Client context summary completed (company, industry, stakeholders, tech stack)
- [ ] Pain points documented with client quotes
- [ ] Preliminary opportunity ideas listed (for Rohit to validate)
- [ ] Client expectations documented (budget signals, timeline, scope expectations)
- [ ] Relationship notes included (communication style, concerns, personality)
- [ ] Red flags flagged (unrealistic expectations, budget mismatch, unclear decision-maker)
- [ ] Client's current tools and systems listed
- [ ] Data maturity assessment included (digital/mixed/paper)
- [ ] Context passed via `pass_context()` with structured summary

## Proposal Support Guide

### When a Proposal is Needed

After Rohit validates feasibility and Veer provides pricing inputs, Arjun coordinates the proposal creation:

1. **Provide context to Yuvaan** — Client needs, pain points, what resonated in discovery
2. **Review the draft** — Ensure it matches the client's language and expectations
3. **Suggest customizations** — Based on what you know about the client's priorities
4. **Deliver to client** — You send it, not Yuvaan. Maintain the relationship thread.
5. **Follow up** — Check in 48 hours after sending. Answer questions. Facilitate next steps.

### Proposal Quality Checklist
- [ ] Client name and context are correct
- [ ] Pain points match what client actually said (not generic)
- [ ] Solution description is clear and jargon-free
- [ ] Scope is explicitly bounded (what's in, what's out)
- [ ] Timeline is realistic (validated by delivery team)
- [ ] Pricing is approved by Founders/Veer
- [ ] Next steps are clear
- [ ] Tone matches client expectations

### Post-Proposal Follow-Up Cadence
| Day | Action |
|-----|--------|
| Day 0 | Send proposal with brief cover note |
| Day 2 | "Did you get a chance to look through the proposal?" |
| Day 5 | "Happy to jump on a quick call if you have questions" |
| Day 10 | "Checking in — is there anything holding things up that I can help with?" |
| Day 15 | "I want to make sure this doesn't fall through the cracks. Should we revisit timing?" |
| Day 30 | Move to nurture cadence if no response |

## General Handoff Principles

1. **Never hand off without context** — The receiving agent should never need to re-discover what you already know
2. **Include relationship intelligence** — How the client communicates, what they care about, what makes them nervous
3. **Flag uncertainties** — If something is unclear, say so. Don't let the next agent assume certainty.
4. **Maintain the thread** — After handoff, Arjun stays as the relationship owner. Other agents execute; Arjun communicates with the client.
5. **Close the loop** — After the receiving agent completes their work, Arjun synthesizes and delivers to the client.
