## 4. Upsell and Cross-Sell Pathways

### Upsell Pathways (Within Same Vertical)

```
Starter → Growth → Enterprise (natural progression within any ICP)
```

| Current State | Upsell Trigger | Upsell Target | Timing |
|--------------|----------------|---------------|--------|
| Single pre-built module | Client reports time savings; asks "what else?" | Multi-workflow automation (S3) | Month 2-3 |
| Basic automation | Client scales operations; new processes emerge | Full workflow suite (S3) | Month 3-6 |
| Growth retainer | Usage exceeds included hours for 2+ months | Enterprise retainer | Month 4-6 |
| Single ICP solution | Client expands to new business line | Replicate solution for new division | Month 6-12 |

### Cross-Sell Pathways (Across Service Verticals)

| Entry Point | Cross-Sell #1 | Cross-Sell #2 | Cross-Sell #3 |
|------------|---------------|---------------|---------------|
| S3: Automation Dev | S1: Team training on the automation | S5: Maintenance retainer | S2: Strategy for next phase |
| S4: Pre-Built | S3: Custom automation for gaps pre-built doesn't cover | S1: User training | S5: Ongoing support |
| S2: Consulting | S3: Build what the roadmap recommends | S1: AI literacy training | S4: Quick wins during build |
| S1: Education | S4: Pre-built tools for processes taught in training | S3: Custom build for advanced needs | S2: Strategic review |
| S5: Maintenance | S3: New automation during quarterly review | S1: Refresh training for new team members | S2: Annual strategy update |

### Cross-Sell by ICP -- Most Likely Sequences

#### SaaS Startups
```
S2 (Consulting) → S3 (Automation Dev) → S1 (Training) → S5 (Maintenance)
    ↗ S4 (Pre-Built quick wins during S3 build)
```

#### Logistics SMEs
```
S4 (Pre-Built) → S1 (Training) → S3 (Automation Dev) → S5 (Maintenance)
    ↗ S2 (Consulting when they're ready for transformation)
```

#### Fintech
```
S2 (Consulting) → S3 (Automation Dev) → S5 (Maintenance) → S1 (Training)
    [Pre-Built rarely applicable due to regulatory customisation needs]
```

#### Professional Services
```
S4 (CA AI Agent) → S1 (EduFlow) → S3 (Custom Automation) → S5 (Maintenance)
    ↗ S2 (Consulting for firm-wide strategy)
```

### Retainer Duration Benchmarks

| ICP | Average Retainer Duration | Retention Rate |
|-----|--------------------------|----------------|
| SaaS Startups | 12-18 months | 75-80% |
| Logistics SMEs | 12-24 months | 80-85% |
| Fintech | 18-36 months | 85-90% |

### Why Duration Varies by ICP

- **SaaS Startups:** Fast movers, tech-savvy. Quick implementation, but may churn if they build internally.
- **Logistics SMEs:** Slower adoption, more training needed. But once embedded, highly sticky.
- **Fintech:** Compliance requirements extend timelines. Regulatory stickiness drives long retainers.
- **Professional Services:** Simpler implementations, but smaller budgets and lower switching costs.

## 7. Service Dependency Map

Some services are prerequisites or strong complements to others.

```
S2 (Consulting)
 ├── ENABLES → S3 (Automation Dev) -- strategy informs what to build
 └── ENABLES → S1 (Education) -- roadmap identifies training needs

S3 (Automation Dev)
 ├── REQUIRES → S5 (Maintenance) -- everything built needs ongoing care
 └── STRONGLY RECOMMENDS → S1 (Education) -- teams must be trained on new systems

S4 (Pre-Built)
 ├── LEADS TO → S3 (Automation Dev) -- pre-built shows value, custom fills gaps
 └── PAIRS WITH → S1 (Education) -- users need onboarding

S1 (Education)
 └── ENHANCES → ALL -- trained teams get more value from every service

S5 (Maintenance)
 └── SUSTAINS → S3, S4 -- deployed systems need monitoring and updates
```

### Mandatory Bundles (Do Not Sell Without)

| Primary Service | Mandatory Bundle | Reason |
|----------------|-----------------|--------|
| S3: Automation Dev (any custom build) | S5: Maintenance (minimum Starter retainer) | Systems degrade without monitoring; client satisfaction at risk |
| S3: Automation Dev (enterprise) | S1: Education (at least basic training) | Enterprise deployments fail without user adoption |
| S4: Pre-Built (3+ modules) | S1: Education (onboarding session) | Multiple modules need structured training |

### Recommended Bundles (Strongly Suggest)

| Primary Service | Recommended Bundle | Revenue Impact |
|----------------|-------------------|----------------|
| S2: Consulting | S3: Automation Dev (to execute the roadmap) | +₹2L - ₹8L implementation |
| S3: Automation Dev | S1: Education (user training) | +₹50K - ₹2L implementation |
| S4: Pre-Built | S3: Automation Dev (for customisation gaps) | +₹1L - ₹3L implementation |
| Any service | S5: Maintenance (Growth retainer upgrade) | +₹25K/month in MRR |
