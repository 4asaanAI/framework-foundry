# Layaa AI — Sales Playbook

## Battle Cards by Persona

### Card 1: Overwhelmed SME Founder (10-50 employees)
- **Pain:** Drowning in ops, can't hire for every role, growth limited by personal capacity
- **Message:** "Get your time back to focus on growing your business, not managing operations."
- **Supporting:** Built for SMEs (not enterprise pricing), we operate it (no IT burden), fast implementation (weeks not months)
- **Best fit:** Starter or Growth package, quick-win workflows first
- **Top objections:** "Too expensive" → compare to hiring cost; "No time to implement" → we handle everything; "Not sure it'll work" → proof-of-concept approach

### Card 2: Operations Manager (20-100 employees)
- **Pain:** Manual processes don't scale, team coordination overhead, quality consistency issues
- **Message:** "Scale your operations without scaling complexity or headcount."
- **Supporting:** Custom automation for specific processes, consistent quality, reduced errors
- **Best fit:** Growth or Enterprise, multi-workflow automation, cross-department coordination
- **Top objections:** "Need to see it first" → pilot approach; "Integration concerns" → custom-built for your tools

### Card 3: Finance/Admin Head (15-75 employees)
- **Pain:** Repetitive data entry, invoice processing, manual reconciliation
- **Message:** "Automate repetitive admin work so you can focus on strategic finance operations."
- **Best fit:** Starter or Growth, invoice/expense/report automation

### Card 4: Department Head (Sales/Marketing/CS)
- **Pain:** Lead/customer management overhead
- **Message:** "Automate follow-ups, reporting, coordination"
- **Best fit:** Growth, CRM automation, customer workflows

### Card 5: Tech-Savvy Business Owner
- **Pain:** Wants automation but no time to DIY
- **Message:** "We build what you'd build yourself, operate it for you"
- **Best fit:** Any package, complex integrations

## Objection Handling Quick Reference

### Pricing Objections
| Objection | Reframe | Escalation |
|-----------|---------|-----------|
| "Too expensive" | Cost of inaction vs. investment. Compare to hiring cost. | None |
| "Can we get a discount?" | Value and flexibility, not price reduction. Suggest phased approach. | → Veer (always) |
| "Need to see ROI first" | Start with single high-impact workflow. 50/50 payment protects them. | None |

### Timing Objections
| "Not ready yet" | Understand root cause (budget/bandwidth/priority). Offer to nurture. | None |
| "Start in X months" | Quantify cost of delay. Lock in date. | None |

### Capability Objections
| "Handle our specific workflow?" | Custom-build approach. Walk through current process. | → Rohit (feasibility) |
| "Integrate with [tool]?" | Most integrations feasible. Verify with tech team in 24h. | → Rohit |
| "Unique requirements" | Strength = custom solutions. Schedule discovery session. | → Rohit (if complex) |

### Trust Objections
| "Haven't heard of you" | Focused on Indian SMEs. Direct founder access. Start small. | → Kabir (if need references) |
| "How do we know it works?" | Free discovery, 50/50 payment, month-to-month retainer. | None |

### Competition Objections
| "Comparing with [competitor]" | Custom vs. template, we operate vs. you maintain, SME pricing. | → Kabir (intel needed) |
| "Just use ChatGPT/Zapier?" | Those = DIY tools. We = your automation team. | None |

## Sales Process Flow
1. **Prospect identified** → Research using ICP profiles
2. **Initial outreach** → Use persona-specific messaging from battle cards
3. **Discovery call** → Use Rohit's 5-phase framework (Business Context → Current State → Requirements → Feasibility → Solution Design)
4. **Proposal** → Use Veer's pricing engine + Yuvaan's templates
5. **Negotiation** → Use objection handling playbook, escalate pricing to Veer
6. **Close** → Standard terms (50% advance, 50% on delivery), escalate non-standard to Abhay
7. **Onboard** → Hand off to delivery team (Rohit → Ujjwal)

## Key Rules
- NEVER offer unauthorized discounts (always escalate to Veer)
- NEVER make ROI guarantees without validated evidence
- NEVER discuss contract terms beyond standard framework (escalate to Abhay)
- NEVER promise specific timelines without technical validation
- Use messaging FRAMEWORKS, not scripts — customize for each prospect
